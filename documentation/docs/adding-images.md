# Adding Images

The memory game selects card images from a local asset folder and a manually maintained filename list. This page explains how the selection works and how to add your own images to the game.

## How Image Selection Works

Card images are managed by two utility functions: `getCardImageNames` and `getCards`.

### `getCardImageNames(cardCount, cardsToMatch)`

This function determines which images are used for a given game session:

1. **Calculates the number of unique pairs** needed: `cardCount / cardsToMatch`.
2. **Shuffles** the full `file_names` array so a different subset is chosen each game.
3. **Slices** the shuffled list down to the number of required pairs.
4. **Duplicates** each image `cardsToMatch` times to create the full set of matching cards.
5. **Shuffles** the final card list again to randomize positions on the board.

### `getCards(cardCount, cardsToMatch)`

Wraps `getCardImageNames` and maps each image name to a full card data object:

| Field | Description |
| :--- | :--- |
| `cardImageName` | The filename, used to load the image from `./src/assets/cards/`. |
| `position` | 1-based index of the card in the initial deck. |
| `cardId` | A unique ID generated by `get_uuid()`, used to track flip and match state. |

## Adding Your Own Images

Follow these two steps to add a new image to the game pool:

### Step 1 — Add the image file

Place your image in the following folder:

```
./src/assets/cards/
```

Any common web image format is supported (`png`, `jpg`, `jpeg`, `webp`, etc.).

### Step 2 — Register the filename

Open `./src/utils/getCardImageNames.js` and add your filename to the `file_names` array:

```js
let file_names = [
    "bug.png",
    "dih_to_yo.jpg",
    // ...existing entries...
    "your-image.png", // ← add your file here
];
```

The filename must match the file in `./src/assets/cards/` exactly, including the extension.

That's it. The game will automatically include your image in the random selection pool from the next session onwards.

:::note
The game only uses as many images as there are unique pairs needed for the session. If the pool has more images than required, the extras are discarded after shuffling. This means adding more images increases variety but doesn't guarantee every image appears in every game.
:::

:::caution
If the total number of images in `file_names` is less than the number of pairs required by the current settings (`cardCount / cardsToMatch`), the game will not have enough unique images to fill the board. Make sure the pool always has at least as many images as the maximum number of pairs your settings allow.
:::
