# useGameControl

`useGameControl` manages all **game session lifecycle transitions** — starting, stopping, resuming, resetting, and ending a game. It coordinates across the board, settings, score, timer, and results stores to ensure every transition leaves the application in a consistent state.

It is consumed primarily by [`useBoard`](./use-board.md), which calls `startGame` on mount and uses the other transitions in response to game events.

## Returns

| Name | Type | Description |
| :--- | :--- | :--- |
| `startGame` | `function` | Initializes or resumes a game session. Syncs settings, generates cards, and starts the timer. |
| `stopGame` | `function` | Pauses the session by locking the board, stopping the timer, and opening the results modal. |
| `resumeGame` | `function` | Re-opens a paused session by unlocking the board and hiding the results modal. |
| `endGame` | `function` | Persists the result to the results store, then calls `stopGame`. |
| `resetGame` | `function` | Fully wipes all in-progress state — cards, flips, score, timer, and modal — to prepare for a fresh game. |

## Store Dependencies

| Store | Fields Read | Fields Written |
| :--- | :--- | :--- |
| `useBoardStore` | `cards`, `matchedCards` | `setCards`, `setMatchedCards`, `setFlippedCards`, `setCardsToMatchIds`, `setIsGameResultsModalOpen` |
| `useSettingsStore` | `cardsToMatch`, `isMoveLimited`, `moveLimit`, `settingsSnapshot` | `setIsGameGoing`, `setIsBoardLocked`, `syncSettingsSnapshot` |
| `useScoreStore` | `moves`, `percentage` | `resetScore` |
| `useTimerStore` | — | `start`, `stop`, `reset` |
| `useResultsStore` | — | `addResult` |

## Transition Reference

### `startGame`

Starts a new game, or restarts after a previous one has ended. If `matchedCards` is non-empty (i.e. a game just finished), a 500ms delay is applied to allow exit animations to complete before resetting.

Key steps:
1. If no game is currently in progress, calls `syncSettingsSnapshot` to lock in the current settings for the session.
2. Generates a fresh card set via `getCards` using the snapshotted `cardCount` and `cardsToMatch`.
3. Unlocks the board, starts the timer, and sets `isGameGoing` to `true`.

:::note
`startGame` reads from `useSettingsStore.getState()` directly (outside React's render cycle) rather than from subscribed state. This ensures it always uses the latest values at the moment it runs, even when called inside a `setTimeout`.
:::

### `stopGame`

Opens the game results modal, locks the board, marks the game as not going, and stops the timer. Used both by `endGame` (after recording the result) and as a standalone pause mechanism.

### `resumeGame`

Hides the results modal, unlocks the board, and sets `isGameGoing` back to `true`. Does not restart the timer or regenerate cards — the session continues exactly where it left off.

### `endGame`

Persists a result entry to `useResultsStore` before stopping the game. The saved result includes:

| Field | Source |
| :--- | :--- |
| `id` | Generated by `get_uuid()` |
| `cards` | Current board cards |
| `moves` | Move count from `useScoreStore` |
| `isMoveLimited` | From `useSettingsStore` |
| `moveLimit` | From `useSettingsStore` |
| `cardsToMatch` | From `useSettingsStore` |
| `percentage` | Match percentage from `useScoreStore` |

### `resetGame`

Clears all transient game state without starting a new game. Intended to be called before `startGame` when the player wants a completely fresh session.

Resets: `flippedCards`, `matchedCards`, `cardsToMatchIds`, `isGameResultsModalOpen`, `score`, and `timer`.

## Usage

`useGameControl` is called inside [`useBoard`](./use-board.md):

```js
const { startGame, stopGame, resumeGame, endGame } = useGameControl();

useEffect(startGame, []);
```
